class QTree.QTree(head_id)[source]

Bases: object

A QTree (=Quantum Tree) is made of a normal tree, a Loop object containing the Qubit objects and the circuit of the problem and a map qvertices that map the vertices id to the corresponding Loop.Qbbit object.

Parameters:

  • loop (Loop) – the Loop object associated to the Quantum Tree

  • qvertices (a dictionary where keys are int and values are Loop.Qbit object) – a dictionary that maps the vertices id to the corresponding Qubit objects

  • last (int) – the id of the last vertex in the photonic circuit

add_edge(parent, leaf, sink=True)[source]

Add an edge between the qubits with index parent and leaf. The qubit with index leaf will be newly created. The qbit parent must be already present in the tree.

Parameters:

  • parent (int) – index of the parent qubit

  • leaf (int) – index of the leaf qubit

  • sink (bool, optional) – you can decide to perform the fusion with or without sinking the parent qubit to the bottom of the circuit, defaults to True

QTree.build_optimal(node, qtree)[source]

Build the optimal DFS-ordered circuit on the object qtree. The idea is to order each vertex’s children, recursively computing the weight of the subtrees. To compute the weight of the subtree we lunch the function on a newly created QTree object.

Parameters:

  • node (TreeNode) – the head of the subtree we are building

  • qtree (QTree) – the QTree object in which we are building the circuit

Returns:
